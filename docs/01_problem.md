# Problem Statement

Modern AI systems can perform impressively across many tasks, but public evaluation often struggles to distinguish durable understanding from pattern completion, memorized exposure, benchmark leakage, or brittle instruction following.

CORE is motivated by a simple question:

> Can a learning architecture be evaluated by its progression toward structured mastery rather than by isolated task performance alone?

## The Evaluation Gap

Many benchmark scores compress model behavior into a single number. That can be useful, but it often hides the most important questions:

- Did the system acquire reusable structure?
- Can it compose primitives into higher-order reasoning?
- Does performance persist across transformed tasks?
- Can it explain, repair, and extend what it appears to know?
- Can improvement be measured as learning rather than prompt-specific adaptation?

CORE focuses on this gap.

## The Mastery Problem

A system that has mastered a domain should do more than answer familiar questions. It should show signs of reusable internal organization:

- stable primitives
- coherent definitions
- robust transfer
- controlled generalization
- resistance to superficial perturbation
- ability to reason from fundamentals

CORE investigates public ways to measure these properties without exposing private implementation details.

## Public Scope

This repository frames the problem and records public evidence. It does not disclose the private mechanism by which CORE attempts to address the problem.

## Current Public Position

CORE is an active experimental project. Public claims should be treated as provisional until supported by repeatable public evidence. This repository is designed to make that evidence trail disciplined from the beginning.
