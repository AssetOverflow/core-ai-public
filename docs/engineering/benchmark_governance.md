# Benchmark Governance

CORE treats benchmark governance as a core engineering responsibility.

Benchmark governance refers to the policies and practices used to preserve:

- benchmark integrity
- evaluation reliability
- reproducibility
- measurement quality
- public trust

## Why It Matters

Weak benchmark governance can create:

- misleading comparisons
- benchmark leakage
- hidden cherry-picking
- unstable metrics
- evaluation contamination
- false confidence

In high-visibility AI systems, this can distort both engineering decisions and public perception.

## CORE Public Principles

CORE publicly emphasizes:

- measurable evidence
- repeatability
- negative controls
- transparent limitations
- careful metric interpretation
- evaluation hardening

## Public Reporting Standards

Public benchmark summaries should:

- define the benchmark category clearly
- define the metric direction clearly
- distinguish goals from proven outcomes
- acknowledge limitations
- avoid sensational framing

## Important Boundary

This repository intentionally withholds:

- benchmark generators
- private datasets
- hidden prompts
- scoring internals
- implementation-level mechanisms

Public credibility without benchmark compromise.
