# Known Limitations and Open Questions

CORE is under active private development.

Public materials in this repository should be interpreted accordingly.

## Current Limitations

At this stage:

- many evaluations remain internal
- public benchmark summaries are still limited
- implementation details are intentionally withheld
- independent external validation has not yet been broadly established
- some evaluation protocols are still evolving
- scalability characteristics are still being studied

## Important Non-Claims

This repository does not currently claim that CORE:

- has solved hallucinations globally
- guarantees deterministic outputs in all settings
- eliminates context limitations
- has solved long-term memory universally
- universally outperforms frontier models
- is production-ready for all domains
- is a finished system

## Open Questions

Important open questions include:

- how far reliability can be improved architecturally
- how retention and transfer scale
- how memory efficiency tradeoffs evolve
- how repeatability interacts with capability
- how evaluation integrity can be preserved at scale
- how long-horizon reasoning can remain stable

## Why This Document Exists

Strong engineering culture requires:

- acknowledging uncertainty
- defining boundaries clearly
- avoiding exaggerated claims
- distinguishing goals from proven outcomes

The purpose of this repository is to build long-term credibility through disciplined evidence.
